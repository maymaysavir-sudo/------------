# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

This is **not a software project** — there is no code, build system, tests, or dependencies. It is a content and business workspace for **סטודיו מאיסה** (Maisa's Studio), a stills photography studio in Tel Aviv. Work here is writing, planning, and content creation in Hebrew, not programming.

The whole repository is currently two files:
- [סטודיו מאיסה.md](סטודיו מאיסה.md) — the source-of-truth business profile (owner details, services, clients, and active work areas).
- `CLAUDE.md` — this file.

## Language

All content is in **Hebrew**. Write deliverables (posts, templates, client messages, plans) in Hebrew and right-to-left unless explicitly asked otherwise. Keep terminology consistent with [סטודיו מאיסה.md](סטודיו מאיסה.md).

## Domain context (from the business profile)

- **Primary specialty:** בוקים לבת מצווה — photo books for girls turning 12, shot in-studio and occasionally outdoors.
- **Other services:** צילומי תדמית (business portraits), צילומי משפחות, and out-of-studio shoots (events, performances, retreats).
- **Preferred audience:** 12-year-old girls for bat-mitzvah books.

## Active work areas (where help is expected)

These come straight from [סטודיו מאיסה.md](סטודיו מאיסה.md) and define the kinds of tasks to expect:

1. **שיווק ומיתוג** — a website that started in Lovable, Instagram posts/reels, paid lead-gen campaigns.
2. **שירות לקוחות** — automating Instagram/WhatsApp replies and building staged client-communication templates (first contact → shoot-day prep file → full process to delivery).
3. **פוסט פרודקשן** — streamlining edits; Lightroom workflow (culling, color, retouching).
4. **הפקות וסדנאות** — concentrated shoot days and workshops for photographers and business owners.

## Working conventions

- Treat [סטודיו מאיסה.md](סטודיו מאיסה.md) as the canonical brief. When the business details change, update it there rather than duplicating facts elsewhere.
- New deliverables should be added as Markdown files; keep file and section names in Hebrew to match the existing style.
